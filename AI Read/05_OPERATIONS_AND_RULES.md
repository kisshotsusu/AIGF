# 启动、验证与维护规则

## 启动入口

- 直播管理页：`启动管理页面.bat`
- 角色管理器：`启动角色管理器.bat`
- 家庭桌宠：`HomeAgent\启动家庭Agent.bat`
- 环境安装：`set_env.bat`
- 模型下载：`down_model.bat`

批处理默认隐藏控制台，仅保留 GUI。端口 9888 被占用通常表示管理器已运行，不要重复启动。

## 修改后的最低验证

完整测试矩阵、单用例命令、真实冒烟和重启验收统一维护在 `08_TESTING.md`；本节只保留最低门槛。

- Python：从项目根目录使用 `.venv\Scripts\python.exe -m py_compile ...`。
- YAML：用 `yaml.safe_load` 读取根配置和 HomeAgent 配置。
- MCP：检查 8765 监听，然后初始化会话并 `list_tools`。
- 网页 Agent：运行 `Skill/web-agent-operator/scripts/web_agent.py` 做无截图端到端测试。
- 直播语音：运行 `.venv\Scripts\python.exe -m unittest discover -s modules/live/tests -v`，至少覆盖重试、冷却和重复进场。
- GUI 设置：特别检查控件不会被可扩展表格或图片挤出窗口。

## 配置写入规则

- 写入前读取最新磁盘配置，只改负责的字段。
- CharacterManager 负责 `vision_mcp.gui_enabled/preload_model`；旧 HomeAgent UI 不得覆盖整段视觉配置。
- 禁用 GUI 时立即停止旧视觉模型进程；`preload_model: false` 表示允许视觉但启动服务时不加载模型，两者不要混淆。
- MCP 配置同步到 `workspace/MCP_SERVERS.yaml`、WorkBuddy 和 Codex 注册表时保留未修改项。
- **`HomeAgent/config.yaml` ↔ `config.d/*.yaml` 由 `CharacterService` 按修改时间双向同步**：主配置 `mtime` 新则覆盖拆分文件，反之用拆分文件回填主配置并落盘；`save_config_section` 同时写两处。因此手改其一后，下次经服务读取可能被另一处旧值覆盖——改配置应走角色工作台或在两处一并修改（详见 `07`）。

## Agent 行为规则

- 用户描述的是最终目标，不是第一步。打开网站后还有搜索/选择/播放时必须继续并验证。
- **禁止任务类型硬编码。** 用户消息的任务/非任务、领域、站点、对象、参数、步骤、成功条件和下一步操作全部由大模型理解与规划；禁止新增关键词表、正则、固定短语、`if 用户文本包含…`、站点专用分支或旧组合流程替代模型判断。
- **禁止手写任务解析逻辑。** 本地代码只校验模型输出是否为合法字段和工具参数，不能根据原始用户文本重新分类、补写搜索词、拆分动作、覆盖模型计划或把某句话强制路由到某个业务执行器。
- **工具只负责操作并返回结果。** 工具不得理解完整任务、决定业务是否完成或规划下一步；模型读取工具结果、时间戳和错误后决定继续观察、点击、输入、重试、换工具或结束。
- 进程启动、实例锁、用户主动停止当前任务、直接重启接力等程序生命周期能力可以保持确定性，但不得扩展为普通业务任务分类器。
- 网页任务优先 `web-agent-operator` 和 DOM 工具。
- 登录态网页优先现有浏览器与 DOM；无 CDP 时保留浏览器并回退窗口视觉，不得为登录任务启动临时 Chrome。
- 图像 GUI 禁用时，不回退截图、窗口图像或 GUI-Actor；仅 `preload_model` 关闭时可以按需懒加载。
- 每个点击、输入、快捷键或滚动后读取操作后截图证据；页面变化不等同于业务完成，仍要验证 URL、文本或媒体状态。
- 操作可停止；HomeAgent 右键和对话条均有“停止当前任务”。
- 不虚构成功。只有验证最终 URL/状态或媒体实际播放后才能汇报完成。
- 执行类任务在本地终态验证后，还要把工具证据交给 MiMo 独立完成检查；失败原因回灌执行循环，证据不足或核验不可用（`fail_closed: true`）时不得报告完成。

## Tool 工具设计与返回契约

- 工具必须是可组合的原子能力，例如列窗口、激活窗口、截图分析、点击、双击、输入、快捷键、读写文件、执行命令；禁止重新增加“搜索并播放”“识别任务后自动执行整套流程”等把任务理解写进工具的组合接口。
- 工具描述只说明能力、输入和机械副作用，不得包含“遇到某类用户请求必须调用”“某站点固定先做什么”等任务路由规则。模型根据计划自行选择工具和调用顺序。
- 参数必须由模型明确提供。工具可以做类型、必填字段、路径存在性和调用格式校验，但不得从用户原文关键词猜参数，也不得静默填入歌曲名、站点名、窗口标题、文件路径或搜索内容。
- 所有工具统一返回结构化结果：`ok/status`、实际执行对象、`observation/result`、错误信息，以及适用的 `task_submitted_at`、`tool_submitted_at`、`screenshot_captured_at/observed_at`、`tool_completed_at`、`tool_sequence`。
- 操作工具只报告实际发生的机械事实，例如点击坐标、输入长度、目标窗口、标题变化和画面变化；“是否满足用户目标”由大模型结合任务计划和最新证据判断。
- 视觉结果必须携带截图时间。截图后发生新的点击、输入、快捷键、媒体命令或明显窗口变化时，旧结果标记过期并从完成证据中淘汰；不得让工具用旧截图替模型决定当前状态。
- 工具失败应返回真实错误和可诊断字段，不得伪装成功，也不得替模型生成重试方案、业务步骤或固定下一操作。
- 每轮必须把原始结构化工具结果按对应 `tool_call_id` 回传模型。模型负责解释结果和决定下一次 tool call；本地循环不得插入站点专用的手写状态机。
- 完成核验只消费经过时间排序和过期淘汰的工具证据；核验模型判断任务是否完成，本地代码不得再用窗口标题必须变化、固定按钮、固定候选点或关键词命中作为业务成功硬门槛。

## 代码与 AI Read 同步规则

- 每次修改代码或配置，都必须在同一次任务里重写对应说明：架构变化改 `01`，文件/职责变化改 `02`，接口/配置变化改 `03`，数据生命周期变化改 `04`，操作与约束变化改 `05`，当前能力与验证结果改 `06`，函数契约和已知坑变化改 `07`，测试数量、命令和流程变化改 `08`。
- “重写对应部分”是修正现有事实、删除过期描述并补足新逻辑，不是只在文件末尾追加日期流水账。
- 新增模块至少同步 `02`；改变控制流至少同步 `01`；新增配置至少同步 `03`；所有影响当前可用能力的改动同步 `06`。README 只在入口、安装、启动或用户使用方式变化时同步。
- `CodeEditorModule` 会追踪实现与 `AI Read`，但不会因为本轮没有修改文档就人为阻断代码验证；模型根据实际影响决定需要更新哪些说明。
- 文档同步后仍要执行实际语法检查和自动测试；文档更新不能替代代码验证。

## 音频

- `audio/` 始终只保留最新 20 个音频，旧文件优先删除。
- 家庭长文本分块：生成一段即入播放队列，同时生成下一段；播放严格按序且不重叠。
- TTS 文本过滤不支持 GBK 的 emoji，避免把音乐符号等字符传给外部批处理。
- 直播语音通过单消费者优先队列串行；欢迎成功后才进入冷却。健康接口忙时不得重复启动 GPT-SoVITS。
## 常驻进程单实例规则（2026-07-21）

- 只能从项目根目录统一启动整套服务；总控制台、HomeAgent、角色管理器、直播、Sound、Vision 使用彼此独立的进程锁。
- Windows 虚拟环境的 `python.exe/pythonw.exe` 可能显示为启动器父进程和系统 Python 子进程，这是一份逻辑实例，不应按两个 PID 强制去重。
- 禁止以“端口暂未打开”作为唯一单实例依据：Vision 在模型预加载期间尚未监听端口，必须先持有进程锁。
- 正常退出和异常终止均由操作系统释放文件锁；锁文件本身可以保留，不能仅凭文件存在判断服务存活。
